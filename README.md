<h2><a id="user-content-i-mở-đầu" class="anchor" aria-hidden="true" href="#i-mở-đầu"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>I. Mở đầu</h2>
<p><code>Git</code> là một phần mềm dùng để quản lý phiên bản của mã nguồn tương tự như <code>SVN</code> nhưng có nhiều ưu điểm hơn, <code>Git</code> đang được sủ dụng rộng rãi hiện nay.
Tuy nhiên trong bài viết này, tôi sẽ nói về git một cách "cá nhân" hơn, mang tính chia sẻ những cái tôi hay làm và hướng tới những người là sysadmin. Mong nhận được ý kiến đóng góp của các bạn.</p>
<h4><a id="user-content-một-số-khái-niệm-cần-làm-rõ" class="anchor" aria-hidden="true" href="#một-số-khái-niệm-cần-làm-rõ"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Một số khái niệm cần làm rõ</h4>
<p><strong><code>Git</code> và <code>Github</code> khác nhau như thế nào?</strong></p>
<p>Lấy ví dụ, bạn có một đoạn script dài 20 dòng, hôm sau bạn tối ưu nó đi, chỉ còn 15 dòng, một ngày khác bạn sửa ở script đó một vài chỗ. Git ghi lại những thời điểm thay đổi đó của bạn và source code của bạn tại thời điểm đó.</p>
<p>Github là một trang web, cho phép bạn lưu source code của mình lên đó. Sự kết hợp hoàn hảo giữa Git và Github mang lại một sự thuận tiện không hề nhỏ cho người dùng. Bạn có thể thay đổi đoạn code của mình mọi lúc mọi nơi mà không sợ bị ghi đè lên hay bị mất dữ liệu do hỏng hóc vì dữ liệu của bạn được lưu cả trên trang web Github và máy cá nhân. Bạn cũng có thể khôi phục được code của mình về một thời điểm bất kỳ nào đó.</p>
<p>Github có bản free và mất phí. Với Github free thì source code của bạn sẽ công khai, có nghĩa là ai cũng có thể xem code của bạn. Nó phù hợp với các phần mềm nguồn mở, và cũng có thể trở thành một blog cá nhân của chính các bạn như các trang blogspot, wordpress,...</p>
<p>Muốn có thể tạo một kho code bí mật của riêng mình thì bạn phải trả phí.</p>
<p>Đối với cá nhân tôi thì github free là quá đủ cho mục đích lưu trữ và chia sẻ thông tin.</p>
<p><strong>Cần phải làm gì để có thể sử dụng <code>Github</code>?</strong></p>
<ul>

</li>
<li>

<li>

</li>
<li>
<p>Push: Đẩy những thay đổi từ máy trạm lên server - tương đương lệnh <code>git push</code></p>
</li>
<li>
<p>Pull: đồng bộ trạng thái từ server về máy trạm - tương đương lệnh <code>git pull</code></p>
</li>
</ul>

<p>Giả sử bạn đang theo dõi repo của tôi và thấy có một số chỗ cần sửa đổi, bạn có thể comment ý kiến của mình vào Repo đó. Sau đó người quản trị sẽ xem xét, thay đổi và trả lời bạn.</p>
<p>Để làm việc này bạn cần vào repo đó, click vào <code>Issue</code>. Ví dụ như hình sau:</p>

<p><a target="_blank" rel="noopener noreferrer" href="https://camo.githubusercontent.com/14f3b5497be1a1ff7ca164be1bde725471f92349/687474703a2f2f692e696d6775722e636f6d2f52546838516a562e706e67"><img src="https://camo.githubusercontent.com/14f3b5497be1a1ff7ca164be1bde725471f92349/687474703a2f2f692e696d6775722e636f6d2f52546838516a562e706e67" data-canonical-src="http://i.imgur.com/RTh8QjV.png" style="max-width:100%;"></a></p>
<p>Lúc này tại Repo của người quản trị sẽ thấy một Issue mới, người quản trị có thể click vào Issue này để xem, sau đó xem xét sửa đổi, comment lại. Khi sửa đổi hoàn tất thì sẽ đóng issue đó lại.</p>
<p><a target="_blank" rel="noopener noreferrer" href="https://camo.githubusercontent.com/3decfc9c6b0ed86b28263af42353d3a1d4cf0e6c/687474703a2f2f692e696d6775722e636f6d2f6a38696f5a67412e706e67"><img src="https://camo.githubusercontent.com/3decfc9c6b0ed86b28263af42353d3a1d4cf0e6c/687474703a2f2f692e696d6775722e636f6d2f6a38696f5a67412e706e67" data-canonical-src="http://i.imgur.com/j8ioZgA.png" style="max-width:100%;"></a></p>
